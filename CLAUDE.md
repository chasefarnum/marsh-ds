# Design System — Claude Instructions

## Architecture

This is a white-label design system factory. Code is always the source of truth. Figma is downstream.

DTCG JSON tokens → Style Dictionary → src/app/globals.css → shadcn CSS vars → 55 components + Figma

Never hand-edit globals.css. It is generated by Style Dictionary from the token files. Any edits will be overwritten on the next build.

## Token Structure

- tokens/primitive/color.tokens.json — Raw brand colors — the only thing that changes per brand
- tokens/primitive/typography.tokens.json — Font families, sizes, weights, line heights
- tokens/primitive/shape.tokens.json — Border radius, shadows
- tokens/primitive/motion.tokens.json — Duration, easing
- tokens/primitive/spacing.tokens.json — Spacing scale, containers, gutters
- tokens/semantic/light.tokens.json — Light mode semantic aliases → primitives
- tokens/semantic/dark.tokens.json — Dark mode semantic aliases → primitives

Semantic tokens map to shadcn CSS variable names (--primary, --background, --accent, etc.) via Style Dictionary.

## Color Token Groups

- color.brand.black/white — Absolute black/white
- color.accent.* — Marsh Blue scale (250/500/750/1000)
- color.neutral.* — Warm grays (250/500/750/1000)
- color.extended.gold/green/purple.* — Extended scales (250/500/750/1000)
- color.extended.red/yellow/success — Status flat colors
- color.overlay.* — Midnight Blue overlays (light/medium/heavy)

## Figma Connection

The Figma file key is stored in .component-contracts. Always update this before doing any Figma work in a new project instance.

To update: "My Figma file key is [KEY] — update .component-contracts with it."

## Component Library

55 components in src/components/ui/. Built on shadcn patterns + Base UI primitives + CVA variants.
Component contracts live in contracts/.

Brand-specific patterns (global nav, global footer, media components, etc.) live in src/components/brand/.

## Skills

- cc-figma-tokens — Sync token files → Figma variable collections
- cc-figma-component — Build a single component in Figma from its contract
- figma-generate-library — Build the full Figma component library
- figma-code-connect-components — Link Figma components to code files
- sync-figma-token — Bidirectional token sync (code ↔ Figma)
- figma-use — Auto-loaded before every Figma write — never skip

## Common Tasks

Rebrand (new client):
"Replace the token files in tokens/primitive/ with [Brand Name] values: [paste colors, fonts, spacing]. Run Style Dictionary to regenerate globals.css. Then sync the updated tokens to Figma."

Add a new component:
"Create a [ComponentName] component. Write the contract first in contracts/, then build the React + TypeScript component in src/components/ui/ using CVA variants and Base UI primitives. Then build it in Figma from the contract and connect it."

Update a token value:
"Update [token name] to [new value] in the token files and regenerate globals.css with Style Dictionary. Then sync the updated tokens to Figma."

Sync tokens to Figma:
"Sync my updated tokens from the token files to Figma."
